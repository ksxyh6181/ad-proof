# 金融凭证系统

本文档介绍 AdProof 项目中的金融凭证系统，该系统使用零知识证明技术实现隐私保护的金融信息验证。

## 概述

金融凭证系统允许用户以保护隐私的方式证明他们的财务状况，而无需披露实际的敏感财务数据。系统支持多种类型的凭证，包括：

1. **收入证明**：证明用户的收入水平，而不泄露具体收入金额
2. **信用评分证明**：证明用户的信用评分范围，而不泄露精确分数
3. **跨境信用证明**：帮助国际学生和移民在新环境中建立信用记录

## 系统架构

金融凭证系统包含以下组件：

### 智能合约

`FinancialCredentialRegistry.sol` 是管理金融凭证的智能合约，提供了凭证的颁发、验证和撤销功能。

### 零知识证明（ZKP）模块

`financial_zkp.rs` 实现了金融凭证的零知识证明电路，包括收入证明、信用评分证明和跨境信用证明。

### 后端服务

金融凭证控制器 `financial_controller.rs` 处理所有与金融凭证相关的API请求，包括：

- 颁发收入证明
- 颁发信用评分证明
- 颁发跨境信用证明
- 验证金融凭证
- 获取和列出金融凭证

### 前端界面

前端界面根据用户角色提供不同的功能：

#### 个人用户

- 查看和管理自己的金融凭证
- 分享凭证哈希给验证方

#### 金融机构

- 颁发收入证明
- 颁发信用评分证明
- 颁发跨境信用证明

#### 验证方（如银行、雇主等）

- 验证用户提供的金融凭证
- 检查凭证是否满足特定要求（如最低收入等级或信用评分）

## 使用流程

### 1. 金融机构颁发凭证

1. 金融机构登录系统，选择需要颁发的凭证类型
2. 输入用户的个人ID和相关信息（如实际收入或信用分数）
3. 系统生成零知识证明，并将凭证哈希存储在区块链上
4. 将凭证信息返回给金融机构，金融机构可以将凭证分享给用户

### 2. 用户管理凭证

1. 用户登录系统，查看自己的金融凭证列表
2. 用户可以查看每个凭证的详细信息和状态（有效/过期）
3. 当需要向验证方证明财务状况时，用户可以分享凭证哈希

### 3. 验证方验证凭证

1. 验证方登录系统，输入用户提供的凭证哈希
2. 选择验证要求（如最低收入等级或信用评分）
3. 系统验证凭证的有效性，并检查是否满足验证方的要求
4. 返回验证结果，而不泄露用户的实际财务数据

## 开发指南

### 添加新的凭证类型

如需添加新的凭证类型，需要修改以下组件：

1. 智能合约：在 `FinancialCredentialRegistry.sol` 中添加新的凭证类型
2. ZKP模块：在 `financial_zkp.rs` 中实现新的电路
3. 后端控制器：在 `financial_controller.rs` 中添加新的API端点
4. 前端界面：为相关角色添加新的界面组件

### 修改验证逻辑

验证逻辑位于 `financial_controller.rs` 的 `verify_financial_credential` 函数中，可以根据需要进行修改。

## API文档

金融凭证系统提供以下API端点：

- **POST /financial/income**：颁发收入证明
- **POST /financial/credit**：颁发信用评分证明
- **POST /financial/cross_border**：颁发跨境信用证明
- **POST /financial/verify**：验证金融凭证
- **POST /financial/get**：获取金融凭证
- **GET /financial/list**：列出所有金融凭证

详细的API参数和返回值请参考API文档。

## 安全考虑

本系统使用零知识证明技术保护用户隐私，但仍需注意以下安全事项：

1. 个人ID应当是匿名的标识符，不应与用户的真实身份直接关联
2. 金融机构应保证其系统的安全，防止未授权访问
3. 用户应妥善保管凭证哈希，避免泄露给未授权的第三方
4. 系统应定期审核和更新，以应对新的安全威胁
